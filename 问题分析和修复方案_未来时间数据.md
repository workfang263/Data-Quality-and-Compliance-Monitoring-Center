# 问题分析和修复方案：未来时间数据

## 🐛 问题描述

**发现时间：** 2025-12-31 10:51

**问题现象：**
1. 当前时间是10:50，但前端显示了16:00的数据
2. 数据库中存在16:00的数据（52订单，$1262.71）
3. 这些数据是在10:35:46写入的

**数据来源：**
- 16:00的数据来自测试店铺（test_shop_*）
- 这些是测试脚本创建的测试数据
- 不应该出现在生产数据库中

---

## 🔍 问题分析

### 根本原因

1. **测试数据污染生产数据库**
   - 测试脚本创建的测试店铺数据被写入了生产数据库
   - 测试店铺（test_shop_*）没有被过滤掉

2. **没有过滤未来时间的订单**
   - 如果API返回的订单时间是未来的时间，应该被过滤掉
   - 当前代码没有检查订单时间是否在未来

3. **数据收集脚本的时间范围计算可能有问题**
   - 可能收集了不应该收集的时间段的数据

---

## ✅ 修复方案

### 方案1：过滤测试店铺（立即修复）

**修改位置：** `database.py` - `get_active_stores()` 方法

**修复内容：**
- 在查询活跃店铺时，过滤掉测试店铺（shop_domain包含'test'的店铺）

**代码修改：**
```python
def get_active_stores(self) -> List[Dict[str, Any]]:
    """获取所有启用的店铺"""
    try:
        with self.get_connection() as conn:
            with conn.cursor() as cursor:
                sql = """
                    SELECT shop_domain, access_token
                    FROM stores
                    WHERE enabled = 1
                      AND shop_domain NOT LIKE '%test%'  -- 过滤测试店铺
                    ORDER BY shop_domain
                """
                cursor.execute(sql)
                return cursor.fetchall()
    except Exception as e:
        logger.error(f"获取活跃店铺失败: {e}")
        return []
```

### 方案2：过滤未来时间的订单（重要修复）

**修改位置：** `data_sync.py` - `sync_store_data()` 方法

**修复内容：**
- 在解析订单时间后，检查订单时间是否在未来
- 如果订单时间在未来，记录警告并跳过该订单

**代码修改：**
```python
# 解析支付时间后
order_dt = ...  # 解析后的订单时间

# ⭐ 新增：过滤未来时间的订单
now = beijing_time()
if order_dt > now:
    order_id = order.get('id', 'unknown')

    )
    continue
```
