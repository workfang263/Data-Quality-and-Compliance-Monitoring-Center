### 2.1 创建时区工具函数文件

创建新文件：`timezone_utils.py`

```python
"""
时区处理工具函数
支持多层级时区配置：店铺/账户级别 > 负责人级别 > 默认UTC+8
"""
import pymysql
from typing import Dict, Any, Optional
from config import DB_CONFIG


def get_db_conn():
    """获取数据库连接"""
    return pymysql.connect(
        host=DB_CONFIG["host"],
        port=DB_CONFIG["port"],
        user=DB_CONFIG["user"],
        password=DB_CONFIG["password"],
        database=DB_CONFIG["database"],
        charset=DB_CONFIG.get("charset", "utf8mb4"),
        cursorclass=pymysql.cursors.DictCursor,
        autocommit=False,
    )


def get_timezone_config(conn, shop_domain: str = None, ad_account_id: str = None, owner: str = None) -> Dict[str, Any]:
    """
    获取时区配置（支持多层级优先级）
    
    优先级：店铺/账户配置 > 负责人配置 > 默认UTC+8
    
    Args:
        conn: 数据库连接
        shop_domain: 店铺域名（可选，用于店铺数据同步）
        ad_account_id: 广告账户ID（可选，用于广告数据同步）
        owner: 负责人名称（可选，用于回退到负责人配置）
    
    Returns:
        {'timezone': 'Asia/Shanghai', 'timezone_offset': 8.0}
    """
    with conn.cursor() as cur:
        # 优先级1：店铺级别配置
        if shop_domain:
            cur.execute("""
                SELECT timezone, timezone_offset 
                FROM store_timezone_mapping 
                WHERE shop_domain = %s
            """, (shop_domain,))
            result = cur.fetchone()
            if result:
                return {
                    'timezone': result['timezone'],
                    'timezone_offset': float(result['timezone_offset'])
                }
        
        # 优先级2：广告账户级别配置
        if ad_account_id:
            cur.execute("""
                SELECT timezone, timezone_offset 
                FROM ad_account_timezone_mapping 
                WHERE ad_account_id = %s
            """, (ad_account_id,))
            result = cur.fetchone()
            if result:
                return {
                    'timezone': result['timezone'],
                    'timezone_offset': float(result['timezone_offset'])
                }
        
        # 优先级3：负责人级别配置
        if owner:
            cur.execute("""
                SELECT timezone, timezone_offset 
                FROM owner_timezone_mapping 
                WHERE owner = %s
            """, (owner,))
            result = cur.fetchone()
            if result:
                return {
                    'timezone': result['timezone'],
                    'timezone_offset': float(result['timezone_offset'])
                }
        
        # 优先级4：默认UTC+8
        return {'timezone': 'Asia/Shanghai', 'timezone_offset': 8.0}


def convert_to_beijing_time(dt, source_timezone_offset: float) -> Any:
    """
    将源时区的时间转换为北京时间（UTC+8）
    
    Args:
        dt: datetime对象（无时区信息）
        source_timezone_offset: 源时区偏移量（小时），例如 -8.0 表示 UTC-8
    
    Returns:
        转换后的datetime对象（北京时间，无时区信息）
    """
    from datetime import timedelta
    
    # 如果源时区已经是UTC+8，不需要转换
    if source_timezone_offset == 8.0:
        return dt
    
    # 计算需要调整的小时数：从源时区到北京时间
    # 例如：UTC-8 到 UTC+8，需要加 16 小时
    hours_to_add = 8.0 - source_timezone_offset
    return dt + timedelta(hours=hours_to_add)


# 为了向后兼容，保留旧的函数名
def get_owner_timezone(conn, owner: str) -> Dict[str, Any]:
    """获取负责人的时区配置（向后兼容）"""
    return get_timezone_config(conn, owner=owner)
```

## ⚠️ 注意事项

### 1. 时区转换计算

**公式：**
```python
hours_to_add = 8.0 - source_timezone_offset
```

**示例：**
- UTC-8 → UTC+8：`8.0 - (-8.0) = 16.0` 小时
- UTC+8 → UTC+8：`8.0 - 8.0 = 0.0` 小时（不需要转换）

### 2. 日期边界处理

- Facebook返回的 `date_start` 是账户时区的日期
- 转换到北京时间后，日期可能会变化（跨天）
- 需要确保过滤逻辑使用转换后的北京时间

### 3. 数据一致性

- **所有数据统一存储为北京时间（UTC+8）**
- **前端不需要做任何修改**
- **时区转换在数据同步时完成**

### 4. 向后兼容

- 没有配置的负责人/店铺/账户自动使用默认值（UTC+8）
- 代码向后兼容，不影响现有功能

### 5. 扩展性

- 如果未来有其他负责人也需要不同的时区，只需在 `owner_timezone_mapping` 表中添加配置
- 如果某个店铺或账户时区特殊，可以单独配置（覆盖负责人配置）

2. ✅ **多层级支持**：支持负责人/店铺/账户三级配置，优先级清晰
3. ✅ **向后兼容**：没有配置的自动使用默认值，不影响现有功能
4. ✅ **数据一致性**：所有数据统一存储为北京时间，前端无需修改
5. ✅ **易于扩展**：支持未来其他负责人或店铺的时区变更

### 关键点

- **时区转换在数据同步时完成**，存储到数据库的数据都是北京时间
- **前端不需要做任何修改**，直接使用数据库中的北京时间数据
- **配置灵活**，支持多层级配置，满足各种需求场景
