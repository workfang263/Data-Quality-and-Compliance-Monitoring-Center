### 1. Shoplazza店铺数据同步（data_sync.py）

**之前的逻辑（第304-326行）：**

```python
# 解析支付时间（placed_at就是支付时间）
# 时间可能是ISO 8601格式（UTC或北京时间）
if 'Z' in placed_at_str:
    # UTC时间，转换为北京时间
    order_dt_utc = datetime.fromisoformat(placed_at_str.replace('Z', '+00:00'))
    order_dt = order_dt_utc.astimezone(pytz.timezone('Asia/Shanghai')).replace(tzinfo=None)
elif '+00:00' in placed_at_str:
    # UTC时间
    order_dt_utc = datetime.fromisoformat(placed_at_str)
    order_dt = order_dt_utc.astimezone(pytz.timezone('Asia/Shanghai')).replace(tzinfo=None)
elif '+08:00' in placed_at_str:
    # 已经是北京时间
    order_dt = datetime.fromisoformat(placed_at_str).replace(tzinfo=None)
else:
    # 没有时区信息，假设是UTC时间，转换为北京时间
    order_dt = order_dt + timedelta(hours=8)
```

**处理逻辑：**
- ✅ 如果时间有 `Z` 或 `+00:00`（UTC时间），转换为北京时间（UTC+8）
- ✅ 如果时间有 `+08:00`（已经是北京时间），直接使用
- ✅ 如果没有时区信息，**假设是UTC时间，加8小时转换为北京时间**

**结论：**
- **所有店铺数据都统一转换为UTC+8（北京时间）存储**
- **没有考虑不同店铺可能有不同时区**
- **假设所有店铺的订单时间都是UTC或UTC+8**

### 2. Facebook广告数据同步（fb_spend_sync.py）

**之前的逻辑（第296-299行）：**

```python
# 解析小时字段：hourly_stats_aggregated_by_advertiser_time_zone = "00:00:00 - 00:59:59"
hour_str = r.get("hourly_stats_aggregated_by_advertiser_time_zone", "")
if hour_str:
    # 提取开始小时，例如 "00:00:00 - 00:59:59" -> "00:00:00"
    hour_start = hour_str.split(" - ")[0] if " - " in hour_str else hour_str
    # 构建 time_hour：日期 + 小时，例如 2025-12-08 00:00:00
    time_hour = datetime.datetime.strptime(
        f"{r['date_start']} {hour_start}",
        "%Y-%m-%d %H:%M:%S"
    )
```

**处理逻辑：**
- ⚠️ 直接将Facebook返回的 `date_start`（账户时区的日期）和 `hour_start`（账户时区的小时）拼接
- ⚠️ **没有进行时区转换**
- ⚠️ **直接当作北京时间存储**

**问题：**
- Facebook返回的 `date_start` 和 `hourly_stats_aggregated_by_advertiser_time_zone` 都是**账户时区**的时间
- 如果账户时区是UTC-8，返回的 "2025-12-16 00:00:00" 实际上是UTC-8的00:00
- 但代码直接当作北京时间（UTC+8）存储，导致时间错位16小时

**结论：**
- **之前所有Facebook账户都假设是UTC+8**
- **如果账户实际时区不是UTC+8，数据会错位**
- **但之前可能所有账户确实都是UTC+8，所以没有发现问题**
