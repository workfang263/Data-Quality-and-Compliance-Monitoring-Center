# Shoplazza 访客浏览量（UV）收集逻辑说明

## 📊 概述

前端8501端口的折线图显示的访客浏览量（UV）数据，是通过Shoplazza的数据分析API获取的。本文档详细说明了数据收集的完整逻辑。

---

## 🔄 数据收集流程

### 1. API接口调用

#### 接口信息
- **接口端点**：`https://{shop_domain}/openapi/2022-01/data/analysis`
- **API版本**：`v2022-01`
- **请求方法**：`GET`

#### 关键参数
- **`begin_time`**：起始时间戳（Unix时间戳，秒）
- **`end_time`**：结束时间戳（Unix时间戳，秒）
- **`tz`**：`8.0`（北京时间 UTC+8）
- **`dt_by`**：`dt_by_day`（**重要：使用天粒度获取按天去重的UV**）
- **`indicator`**：`['uv']`（只获取访客数指标）
- **`filter_crawler_type`**：空字符串 `''`（**不过滤爬虫流量**，包含所有访问）
- **`page`**：页码（从1开始）
- **`limit`**：每页记录数（最大200）

#### API调用示例代码
```python
# 从 data_sync.py 中提取的关键代码
analysis_data_daily = api.get_data_analysis_all_pages(
    begin_ts,  # 起始时间戳
    end_ts,    # 结束时间戳
    'dt_by_day',  # 使用天粒度获取按天去重的UV
    indicator=['uv'],  # 只获取访客数
    filter_crawler_type=''  # 不过滤爬虫流量（空字符串）
)
```

---

### 2. 数据去重逻辑

#### 按天去重（Daily Deduplication）
- **逻辑**：访客数是按**天**去重的，不是按小时去重
- **含义**：同一天内，同一个访客（IP）无论访问多少次，只算作1个访客
- **数据粒度**：使用 `dt_by_day` 参数，返回的是**每天**的访客数

#### 为什么使用天粒度？
1. **Shoplazza API设计**：数据分析API只提供按天去重的UV，没有按小时去重的UV
2. **业务逻辑**：UV的定义通常是"独立访客数"，在同一天内重复访问不算新的访客
3. **数据一致性**：与Shoplazza后台统计逻辑保持一致

---

### 3. 爬虫流量处理

#### 不过滤爬虫流量
- **设置**：`filter_crawler_type` 参数为空字符串 `''` 或 `None`
- **含义**：**不过滤爬虫流量**，统计所有访问（包括爬虫、机器人等）
- **原因**：
  - 与订单数据统计口径保持一致
  - 某些"爬虫"可能是真实用户（如搜索引擎爬虫）
  - 确保数据完整性

#### 注意
- 如果要过滤爬虫流量，可以设置 `filter_crawler_type='official_crawler'`
- 但当前系统设置是不过滤，保持最大数据量

### 4. 数据存储逻辑

#### 写入数据库

##### 单店铺明细表（shoplazza_store_hourly）
```python
# 从 data_sync.py 中的关键逻辑
# 1. 按天获取访客数
daily_visitors = {}  # {day: uv}
for item in analysis_data_daily:
    day_key = item['date_time'].date()
    daily_visitors[day_key] = item['uv']

# 2. 将同一天的所有小时使用相同的访客数
for hour_key in hourly_data.keys():
    day_key = hour_key.date()
    if day_key in daily_visitors:
        hourly_data[hour_key]['uv'] = daily_visitors[day_key]
```

##### 汇总表（shoplazza_overview_hourly）
```sql
-- 使用 MAX 或 GREATEST 函数，确保同一天所有小时使用最大值
INSERT INTO shoplazza_overview_hourly
(time_hour, total_gmv, total_orders, total_visitors, avg_order_value)
VALUES (%s, %s, %s, %s, %s)
ON DUPLICATE KEY UPDATE
    total_gmv = total_gmv + VALUES(total_gmv),
    total_orders = total_orders + VALUES(total_orders),
    total_visitors = GREATEST(total_visitors, VALUES(total_visitors)),  -- 取最大值
    ...
```

#### 关键点
- **同一天所有小时使用相同的访客数**（因为是按天去重的）
- **使用 `GREATEST()` 函数**：如果同一个小时的记录被多次写入，取最大值（虽然理论上应该相同）
- **多店铺汇总**：所有店铺的访客数直接累加（不同店铺的访客是不同的IP）

### 5. 前端显示逻辑

#### 数据库查询

##### 小时模式（Hourly View）
```sql
-- 按小时查询，但同一天所有小时使用相同的访客数
SELECT 
    time_hour,
    SUM(total_gmv) AS total_gmv,
    SUM(total_orders) AS total_orders,
    MAX(total_visitors) AS total_visitors  -- 使用MAX，确保同一天相同
FROM shoplazza_overview_hourly
WHERE ...
GROUP BY time_hour
ORDER BY time_hour ASC
```

##### 天模式（Daily View）
```sql
-- 按天聚合
SELECT 
    DATE(time_hour) as date,
    SUM(total_gmv) as total_gmv,
    SUM(total_orders) as total_orders,
    MAX(total_visitors) as total_visitors  -- 使用MAX
FROM shoplazza_overview_hourly
WHERE ...
GROUP BY DATE(time_hour)
ORDER BY date ASC
```

#### 前端处理（dashboard.py）

##### 小时模式处理
```python
# 修复访客数：确保同一天所有小时的访客数都相同（按天取最大值）
df['date'] = df['time'].dt.date
df['total_visitors'] = df.groupby('date')['total_visitors'].transform('max')
```

##### 显示统计
```python
# 小时模式：按天分组取最大值后累加
if granularity == 'hour':
    total_visitors = df.groupby(df['time'].dt.date)['total_visitors'].max().sum()
else:
    # 天模式：直接累加（每天的数据已经是去重后的）
    total_visitors = df['total_visitors'].sum()
```

#### 指标说明
- **显示名称**：`总访客数 (UV)`
- **单位**：人次（独立访客数）
- **提示信息**：`"所有店铺在该时间段内的独立访客数（不过滤爬虫流量，包含所有访问，按天去重）"`

## 📋 数据流程总结

### 完整流程
```
1. 数据同步脚本（data_sync.py）
   ↓
2. 调用 Shoplazza 数据分析API
   - 参数：dt_by_day（天粒度）
   - 指标：uv（访客数）
   - 不过滤爬虫流量
   ↓
3. 解析API返回数据
   - 按天存储访客数：{day: uv}
   ↓
4. 分配到每小时
   - 同一天的所有小时使用相同的访客数
   ↓
5. 写入数据库
   - 单店铺明细表：shoplazza_store_hourly
   - 汇总表：shoplazza_overview_hourly
   - 使用 GREATEST() 确保数据正确
   ↓
6. 前端查询（dashboard.py）
   - 小时模式：按天分组取最大值后累加
   - 天模式：直接累加
   ↓
7. 显示在折线图中
   - 可勾选显示/隐藏
   - 显示在指标卡片中
```

## ⚠️ 重要说明

### 1. 访客数统计规则
- **按天去重**：同一天内，同一访客（IP）只算1次
- **不过滤爬虫**：包含所有访问（爬虫、机器人等）
- **多店铺累加**：不同店铺的访客直接累加（不同店铺的IP是不同的）

### 2. 数据时间对齐
- **查询时间**：使用 `begin_time` 和 `end_time`（Unix时间戳）
- **统计日期**：根据 `date_time` 字段确定是某一天的访客数
- **时区**：统一使用北京时间（UTC+8）

### 3. 数据一致性
- **同一天所有小时使用相同值**：这是按天去重的必然结果
- **前端处理**：即使数据库中有不同值，前端也会按天取最大值后显示
- **SUM vs MAX**：
  - 小时模式汇总：使用 `MAX(total_visitors)`
  - 天模式汇总：使用 `MAX(total_visitors)` 后再累加

### 4. 数据准确性
- **数据来源**：Shoplazza官方数据分析API
- **数据延迟**：可能有几分钟延迟（与API数据更新频率相关）
- **数据校验**：可以通过 `verify_today_data.py` 和 `verify_yesterday_data.py` 验证数据
