# SQL命令详细解释

## 验证时区配置表是否创建成功

### 命令1：`SHOW TABLES LIKE '%timezone%';`

**命令含义：**
- `SHOW TABLES`：显示当前数据库中的所有表名
- `LIKE '%timezone%'`：筛选条件，只显示表名中包含 "timezone" 的表
  - `%` 是通配符，表示任意字符（0个或多个）
  - `%timezone%` 表示：前面可以有任意字符，中间必须有 "timezone"，后面也可以有任意字符

**作用：**
- 快速查看是否创建了时区相关的表
- 如果创建成功，应该看到3个表：
  - `owner_timezone_mapping`
  - `store_timezone_mapping`
  - `ad_account_timezone_mapping`

**示例结果：**
```
+----------------------------------+
| Tables_in_shoplazza_dashboard    |
+----------------------------------+
| owner_timezone_mapping           |
| store_timezone_mapping           |
| ad_account_timezone_mapping      |
+----------------------------------+
```

---

### 命令2：`DESCRIBE owner_timezone_mapping;`

**命令含义：**
- `DESCRIBE`（或简写为 `DESC`）：描述表的结构
- `owner_timezone_mapping`：要查看的表名

**作用：**
- 显示表的所有字段（列）信息
- 包括：字段名、数据类型、是否允许NULL、键信息、默认值等

**示例结果：**
```
+------------------+--------------+------+-----+-------------------+-------------------+
| Field            | Type         | Null | Key | Default           | Extra             |
+------------------+--------------+------+-----+-------------------+-------------------+
| id               | int          | NO   | PRI | NULL              | auto_increment    |
| owner            | varchar(255) | NO   | UNI | NULL              |                   |
| timezone         | varchar(64)  | NO   |     | NULL              |                   |
| timezone_offset  | decimal(4,1) | NO   |     | NULL              |                   |
| created_at       | timestamp    | YES  |     | CURRENT_TIMESTAMP | DEFAULT_GENERATED |
| updated_at       | timestamp    | YES  |     | CURRENT_TIMESTAMP | DEFAULT_GENERATED |
+------------------+--------------+------+-----+-------------------+-------------------+
```

**字段说明：**
- `Field`：字段名
- `Type`：数据类型（如 int、varchar、decimal、timestamp）
- `Null`：是否允许为空（NO=不允许，YES=允许）
- `Key`：键类型（PRI=主键，UNI=唯一键）
- `Default`：默认值
- `Extra`：额外信息（auto_increment=自动递增）

---

### 命令3：`DESCRIBE store_timezone_mapping;` 和 `DESCRIBE ad_account_timezone_mapping;`

**命令含义：**
- 与上面相同，只是查看另外两个表的结构

**作用：**
- 验证另外两个表是否也正确创建
- 确认表结构是否符合预期

### 命令4：复杂查询（查看表的详细信息）

```sql
SELECT 
    TABLE_NAME as '表名',
    TABLE_COMMENT as '表注释',
    ENGINE as '存储引擎',
    TABLE_ROWS as '记录数'
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = DATABASE()
  AND TABLE_NAME IN (
    'owner_timezone_mapping',
    'store_timezone_mapping',
    'ad_account_timezone_mapping'
  )
ORDER BY TABLE_NAME;
```

**命令分解：**

1. **`SELECT ... FROM information_schema.TABLES`**
   - `information_schema`：MySQL的系统数据库，存储数据库元数据（关于数据库的信息）
   - `TABLES`：系统表，存储所有数据库和表的信息
   - 作用：从系统表中查询表的元数据信息

2. **`TABLE_NAME as '表名'`**
   - `TABLE_NAME`：系统表中的字段，存储表名
   - `as '表名'`：给查询结果列起一个中文别名，显示时更友好

3. **`TABLE_COMMENT as '表注释'`**
   - `TABLE_COMMENT`：表的注释（我们在创建表时用 `COMMENT='...'` 设置的）
   - 作用：显示表的说明信息

4. **`ENGINE as '存储引擎'`**
   - `ENGINE`：表的存储引擎（如 InnoDB、MyISAM）
   - 作用：显示表使用的存储引擎类型

5. **`TABLE_ROWS as '记录数'`**
   - `TABLE_ROWS`：表中大约有多少行数据
   - 作用：显示表的记录数量（新表应该是0）

6. **`WHERE TABLE_SCHEMA = DATABASE()`**
   - `TABLE_SCHEMA`：数据库名
   - `DATABASE()`：当前连接的数据库名
   - 作用：只查询当前数据库的表，不查询其他数据库的表

7. **`AND TABLE_NAME IN (...)`**
   - `IN (...)`：在列表中匹配
   - 作用：只查询这三个特定的表

8. **`ORDER BY TABLE_NAME`**
   - `ORDER BY`：排序
   - 作用：按表名字母顺序排列结果

**示例结果：**
```
+------------------------------+----------------------+--------+----------+
| 表名                         | 表注释               | 存储引擎 | 记录数    |
+------------------------------+----------------------+--------+----------+
| ad_account_timezone_mapping  | 广告账户时区配置表   | InnoDB |        0 |
| owner_timezone_mapping       | 负责人时区配置表     | InnoDB |        0 |
| store_timezone_mapping       | 店铺时区配置表       | InnoDB |        0 |
+------------------------------+----------------------+--------+----------+
```

**作用：**
- 一次性查看三个表的详细信息
- 确认表都正确创建，且注释、存储引擎等都符合预期

---

## 总结

这三个验证命令的作用：

1. **`SHOW TABLES LIKE '%timezone%'`**：快速检查表是否存在
2. **`DESCRIBE 表名`**：详细查看表结构，确认字段是否正确
3. **复杂查询**：查看表的元数据信息，确认表配置是否正确

通过这些验证，可以确保：
- ✅ 三个表都已成功创建
- ✅ 表结构符合预期
- ✅ 表配置正确（存储引擎、注释等）
