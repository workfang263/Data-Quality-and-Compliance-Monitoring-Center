## 🎯 学习模块 1：自动化监控脚本（Watcher）

### 学习目标
- 如何写一个 Python 守护进程
- 如何定期检查数据库 `updated_at` 时间
- 如何通过钉钉/企业微信/邮件发告警

### 实战项目

#### 1. 基础监控脚本 ✅（已完成）
**文件：** `tests/db_watcher.py`

**功能：**
- 检查 `sync_status` 表的最后同步时间
- 超过 20 分钟未更新则发送钉钉告警

**使用方法：**
```bash
# 手动运行
python tests\db_watcher.py

# 定时任务（Windows任务计划程序）
# 创建任务，每5分钟执行一次：run_db_watcher.bat
```

#### 2. 守护进程版本 ✅（已创建）
**文件：** `tests/db_watcher_daemon.py`

**功能：**
- 持续运行，每5分钟自动检查
- 支持优雅退出（Ctrl+C）
- 异常自动恢复

**使用方法：**
```bash
# 直接运行（前台运行，方便调试）
python tests\db_watcher_daemon.py

# 后台运行（Windows）
start /b python tests\db_watcher_daemon.py

# 或者使用任务计划程序，设置为"持续运行"
```

**学习点：**
- `while True` 循环实现持续运行
- `signal.signal()` 实现优雅退出
- `time.sleep()` 实现定期检查
- 异常处理保证稳定性

## 🎯 学习模块 2：数据对账（Data Reconciliation）

### 学习目标
- 如何写 SQL 脚本自动比对"明细表"和"汇总表"的金额是否对得上
- 这是测开的"后端自动化"核心技能

### 实战项目

#### 数据对账脚本 ✅（已创建）
**文件：** `tests/data_reconciliation.py`

**功能：**
1. **小时级对账：** 对比 `shoplazza_store_hourly`（明细表）和 `shoplazza_overview_hourly`（汇总表）
   - 验证：`SUM(明细表.总销售额) == 汇总表.总销售额`
   - 验证：`SUM(明细表.总订单数) == 汇总表.总订单数`

2. **日汇总对账：** 对比按天汇总的数据是否一致

**使用方法：**
```bash
# 对账今天的数据
python tests\data_reconciliation.py

# 对账指定日期的数据
python tests\data_reconciliation.py 2025-12-31
```

**学习点：**
- SQL 聚合函数：`SUM()`, `MAX()`, `GROUP BY`
- 数据一致性验证逻辑
- 差异检测和告警机制

## 🎓 面试常见问题

### Q1: 你怎么保证你的脚本一直在跑？

**答案要点：**
1. **监控脚本：** 定期检查脚本的运行状态（通过检查数据库更新时间）
2. **告警机制：** 超过阈值自动发送钉钉/邮件告警
3. **守护进程：** 使用守护进程模式，异常自动恢复
4. **任务计划：** Windows 任务计划程序自动重启失败的任务

**实战代码：** `db_watcher.py`, `db_watcher_daemon.py`

---

### Q2: 数据错了你怎么第一时间知道？

**答案要点：**
1. **数据对账：** 定期对比明细表和汇总表，发现数据不一致
2. **数据校验：** 对比数据库数据和 API 数据
3. **异常检测：** 检测数据异常波动（例如：销售额突然为0）
4. **告警机制：** 发现问题立即发送告警

**实战代码：** `data_reconciliation.py`

---

### Q3: 你如何保证数据质量？

**答案要点：**
1. **数据对账：** 明细表 vs 汇总表一致性验证
2. **数据校验：** 数据库 vs API 数据一致性验证
3. **异常监控：** 实时监控数据同步状态
4. **自动化测试：** 定期运行数据质量检查脚本

**实战代码：** `data_reconciliation.py`, `verify_today_data.py`
