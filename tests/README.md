# 测试脚本说明

## 测试脚本列表

### 1. test_data_consistency.py - 数据一致性测试

**测试目标：**
- 模拟多个10分钟段的数据同步
- 验证数据库汇总表的数值正确性
- 确保数据不会翻倍

**运行方式：**
```bash
# 使用 pytest 运行
pytest tests/test_data_consistency.py -v -s

# 或者直接运行
python tests/test_data_consistency.py
```

**测试场景：**
1. **多段数据一致性测试**：模拟3个10分钟段，验证数据库汇总表的数值等于3个段的总和
2. **防翻倍测试**：处理同一个10分钟段两次，验证数据不会翻倍

---

### 2. test_concurrent_write.py - 并发写入测试

**测试目标：**
- 模拟多个线程同时写入同一个小时的数据
- 验证数据库的原子累加是否生效
- 验证是否有数据丢失

**运行方式：**
```bash
python tests/test_concurrent_write.py
```

**测试场景：**
- 创建5个线程，同时写入同一个小时的数据
- 验证数据库中的数据等于所有线程数据的总和
- 验证没有数据丢失或重复累加

---

## 测试前准备

### 1. 安装依赖

确保已安装 pytest：
```bash
pip install pytest
```

### 2. 配置数据库

确保 `config.py` 中的数据库配置正确。

### 3. 清理测试数据

测试脚本会自动清理测试数据，但建议在运行测试前：
- 确保测试时间（2025-12-31 15:00:00 和 16:00:00）没有重要数据
- 或者修改测试脚本中的测试时间

---

## 测试结果解读

### 数据一致性测试

**通过标准：**
- 数据库汇总表的数值等于所有段的总和
- 重复执行同一个段，数据会累加（这是增量累加模式的预期行为）

**失败情况：**
- 数据不匹配：说明逻辑有问题
- 数据翻倍：说明累加逻辑有问题

### 并发写入测试

**通过标准：**
- 数据库中的数据等于所有线程数据的总和
- 没有数据丢失
- 没有重复累加

**失败情况：**
- 数据不一致：可能存在数据丢失或重复累加问题
- 线程执行失败：可能存在并发冲突

---

## 注意事项

1. **测试数据清理**：测试脚本会自动清理测试数据，但建议在非生产环境运行
2. **测试时间**：测试脚本使用固定的测试时间（2025-12-31），确保这个时间没有重要数据
3. **数据库连接**：确保数据库连接正常，测试脚本需要连接数据库
4. **Mock 数据**：测试脚本使用 Mock 数据，不会调用真实的 API

---

## 扩展测试

如果需要更全面的测试，可以考虑：

1. **性能测试**：测试大量数据写入的性能
2. **异常测试**：测试异常情况下的数据一致性
3. **边界测试**：测试边界情况（如0订单、负数等）
4. **集成测试**：测试与真实 API 的集成
