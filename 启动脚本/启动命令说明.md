# 系统启动命令说明

本文档说明如何启动新系统和旧系统。

---

## 📋 端口说明

- **旧系统（Streamlit看板）**：端口 8502
- **新系统后端（FastAPI）**：端口 8000
- **新系统前端（Vue）**：端口 5173

---

## 🔧 启动旧系统（Streamlit看板）

### 方法1：双击运行脚本（推荐）

直接双击运行：
```
启动脚本\启动旧系统.bat
```

### 方法2：使用CMD命令行

```cmd
REM 清理端口8502（如果被占用）
启动脚本\kill_port.bat 8502
REM 启动Streamlit看板
streamlit run dashboard.py --server.port 8502 --server.address 0.0.0.0
```

**访问地址：** http://localhost:8502

---

## 🚀 启动新系统（Vue + FastAPI）

### 方法1：双击运行脚本（推荐）

直接双击运行：
```
启动脚本\启动新系统.bat
```

脚本会自动：
- 清理被占用的端口（8000 和 5173）
- 启动后端API服务
- 启动前端服务
- 在独立窗口运行，便于查看日志

### 方法2：使用CMD命令行（需要打开两个终端窗口）

#### 终端1 - 启动后端API（固定端口8000）

```cmd
REM 清理端口8000（如果被占用）
启动脚本\kill_port.bat 8000
REM 启动后端API
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 终端2 - 启动前端（固定端口5173）

```cmd
REM 清理端口5173（如果被占用）
启动脚本\kill_port.bat 5173
REM 切换到frontend目录（重要：必须在这个目录下运行npm命令）
cd frontend
REM 启动前端
npm run dev -- --port 5173
```

**访问地址：**
- 后端API: http://localhost:8000
- API文档: http://localhost:8000/docs
- 前端页面: http://localhost:5173
- **店铺运营（店匠员工归因）**：http://localhost:5173/store-ops（需账号具备 `can_view_store_ops` 权限）

## 店铺运营 · 定时同步（可选）

- **接口**：`POST http://localhost:8000/api/internal/store-ops/sync`，请求头 `X-Internal-Key` 与项目根 `.env` 中 **`STORE_OPS_SYNC_SECRET`** 一致；体可为空 JSON `{}`。返回 **202** 表示任务已入队，实际拉单在后台执行。
- **脚本**：`scripts/store_ops_sync_hourly.ps1`（任务计划直接调此文件）；备份参考见 `scripts/store_ops_sync_hourly.example.ps1`。
- **手动测试**（密钥必须用引号）：  
  `$env:STORE_OPS_SYNC_SECRET="与.env一致"`  
  `powershell -NoProfile -ExecutionPolicy Bypass -File "D:\projects\line chart\scripts\store_ops_sync_hourly.ps1"`
- **HTTPS 校验**：若走代理出现 SSL 报错，可在 `.env` 设置 **`STORE_OPS_HTTPS_VERIFY=false`**（生产环境建议恢复为默认校验）；详见 `backend/README.md`。
- **同步结果落库**：排障可在 MySQL 查表 `store_ops_sync_runs`（迁移：`db/migrations/20260403_store_ops_sync_runs.sql`）。

### Windows 任务计划程序（每小时一次）

1. **先做好**：项目根 `.env` 已配置 **`STORE_OPS_SYNC_SECRET`** 并已重启后端；本机手动执行一次上面「手动测试」成功。
2. **环境变量（推荐）**：`Win + R` → `sysdm.cpl` → **高级** → **环境变量** → 在 **用户变量** 中新建 **`STORE_OPS_SYNC_SECRET`**，值与 `.env` 完全相同（任务以同一 Windows 用户运行时才能读到）。
3. **打开**：`Win + R` → 输入 **`taskschd.msc`** → 回车。
4. **创建任务**：右侧 **创建任务…**（勿用「创建基本任务」，便于设重复间隔）。
5. **常规**：名称如 `店铺运营-店匠同步`；**运行身份**选你平时登录且能跑后端的用户（便于读取用户环境变量）；可勾选 **使用最高权限运行**（按需）。
6. **触发器** → **新建**：**按计划** → **每天** → 开始时间任一整点（如 `00:00:00`）→ **高级设置** 勾选 **重复任务间隔** **1 小时**，**持续时间** **无限期**。
7. **操作** → **新建**：**启动程序**  
   - **程序或脚本**：`powershell.exe`  
   - **添加参数**（路径按你机器修改，注意引号）：  
     `-NoProfile -ExecutionPolicy Bypass -File "D:\projects\line chart\scripts\store_ops_sync_hourly.ps1"`  
   - **起始于**（可选）：`D:\projects\line chart\scripts`
8. **条件**：笔记本可取消「只有交流电源才启动」，避免合盖不跑。
9. **设置**：建议勾选 **过了计划开始时间则立即启动任务**。
10. 确定保存 → 右键该任务 **运行** → **上次运行结果** `0x0` 为成功；后端日志应出现 `POST /api/internal/store-ops/sync` **202**。

**说明**：任务执行时 **uvicorn 必须已在运行**；若后端部署在别的机器，在任务参数中增加：  
`-BaseUrl "http://服务器IP:8000"`（或在脚本内改默认 `$BaseUrl`）。后续若改用进程内 APScheduler，请 **禁用** 本任务，避免重复同步。

## ⚠️ 注意事项

1. **目录位置**
   - 所有命令都需要在项目根目录（`D:\projects\line chart`）下运行
   - 前端命令必须先在 `frontend` 目录下运行

2. **端口占用处理**
   - `kill_port.bat` 会自动查找并终止占用端口的进程
   - 如果无法终止进程（需要管理员权限），请以管理员身份运行CMD
   - 手动查看端口占用：`netstat -ano | findstr ":端口号"`

3. **推荐使用批处理脚本**
   - 批处理脚本（`.bat`文件）会自动处理端口清理
   - 脚本会在新窗口运行服务，便于管理

4. **停止服务**
   - 如果使用批处理脚本启动，关闭对应的服务窗口即可停止
   - 如果使用命令行启动，在对应终端按 `Ctrl+C` 停止

## 🔍 故障排查

### 端口被占用

如果端口被占用，可以使用以下命令手动清理：

```cmd
REM 清理指定端口（例如：清理8000端口）
启动脚本\kill_port.bat 8000
```

### 查看端口占用情况

```cmd
REM 查看所有监听端口
netstat -ano | findstr "LISTENING"

REM 查看特定端口（例如：查看8000端口）
netstat -ano | findstr ":8000"
```

### 前端命令报错：找不到package.json

**错误信息：**
```
npm error enoent Could not read package.json
```

**解决方法：**
确保在 `frontend` 目录下运行 `npm` 命令：
```cmd
cd frontend
npm run dev -- --port 5173
```

### 后端命令报错：找不到模块

**错误信息：**
```
ModuleNotFoundError: No module named 'uvicorn'
```

**解决方法：**
安装后端依赖：
```cmd
cd backend
pip install -r requirements.txt
```

### 前端命令报错：找不到模块

**错误信息：**
```
Error: Cannot find module 'vite'
```

**解决方法：**
安装前端依赖：
```cmd
cd frontend
npm install
```
