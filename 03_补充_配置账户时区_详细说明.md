# 配置账户时区 - SQL命令详细解释

## 为什么需要这一步？

你提到：
- `act_1867789100501535` 是 UTC-8 时区的（新账户）
- `act_1899226041013050` 是 UTC+8 时区的（旧账户）

但是"小一"的负责人级别时区配置是 UTC-8，这会导致：
- 如果只配置负责人级别，两个账户都会使用 UTC-8（不正确）
- 旧账户 `act_1899226041013050` 应该保持 UTC+8

**解决方案：**
- 在账户级别单独配置时区，覆盖负责人配置
- 新账户配置为 UTC-8
- 旧账户配置为 UTC+8

---

## SQL命令说明

### 命令1：为新账户配置UTC-8

```sql
INSERT INTO ad_account_timezone_mapping (ad_account_id, timezone, timezone_offset) 
VALUES ('act_1867789100501535', 'America/Los_Angeles', -8.0)
ON DUPLICATE KEY UPDATE 
  timezone = VALUES(timezone), 
  timezone_offset = VALUES(timezone_offset);
```

**作用：**
- 在 `ad_account_timezone_mapping` 表中为账户 `act_1867789100501535` 配置时区
- 时区：`America/Los_Angeles`（UTC-8）
- 偏移量：`-8.0`
- 这个配置会覆盖负责人级别的配置

### 命令2：为旧账户配置UTC+8

```sql
INSERT INTO ad_account_timezone_mapping (ad_account_id, timezone, timezone_offset) 
VALUES ('act_1899226041013050', 'Asia/Shanghai', 8.0)
ON DUPLICATE KEY UPDATE 
  timezone = VALUES(timezone), 
  timezone_offset = VALUES(timezone_offset);
```

**作用：**
- 在 `ad_account_timezone_mapping` 表中为账户 `act_1899226041013050` 配置时区
- 时区：`Asia/Shanghai`（UTC+8）
- 偏移量：`8.0`
- 这个配置会覆盖负责人级别的配置，确保旧账户使用UTC+8

---

## 时区配置优先级说明

我们的系统支持多层级时区配置，优先级从高到低：

1. **账户级别配置**（最高优先级）
   - 在 `ad_account_timezone_mapping` 表中配置
   - 如果账户有配置，直接使用账户配置

2. **负责人级别配置**（中等优先级）
   - 在 `owner_timezone_mapping` 表中配置
   - 如果账户没有配置，使用负责人的配置

3. **默认配置**（最低优先级）
   - 如果都没有配置，使用默认值 UTC+8

**当前配置结果：**
- `act_1867789100501535`：使用账户级别配置（UTC-8）✅
- `act_1899226041013050`：使用账户级别配置（UTC+8）✅
- 其他账户：如果没有账户级别配置，使用负责人配置或默认值

## 执行后验证

执行完这两个SQL后，运行以下查询验证：

```sql
-- 查看"小一"的所有Facebook广告账户及其时区配置
SELECT 
    a.ad_account_id,
    a.owner,
    COALESCE(atz.timezone, otz.timezone, 'Asia/Shanghai') as timezone,
    COALESCE(atz.timezone_offset, otz.timezone_offset, 8.0) as timezone_offset,
    CASE 
        WHEN atz.ad_account_id IS NOT NULL THEN '账户级别配置'
        WHEN otz.owner IS NOT NULL THEN '负责人级别配置'
        ELSE '默认配置'
    END as config_source
FROM ad_account_owner_mapping a
LEFT JOIN ad_account_timezone_mapping atz ON a.ad_account_id = atz.ad_account_id
LEFT JOIN owner_timezone_mapping otz ON a.owner = otz.owner
WHERE a.owner = '小一';
```

```
| act_1899226041013050 | 小一 | Asia/Shanghai       |           8.0  | 账户级别配置     |
+----------------------+------+---------------------+----------------+------------------+
```
