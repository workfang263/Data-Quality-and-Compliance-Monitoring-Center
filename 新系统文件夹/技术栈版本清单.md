# 技术栈版本清单

## 📋 说明

这个文档记录了项目使用的所有技术栈及其版本号。选择版本的原则：
1. **稳定性优先**：选择经过验证的稳定版本
2. **兼容性**：确保各技术栈之间兼容
3. **功能完整**：满足项目需求
4. **社区支持**：有良好的文档和社区支持

---

## 🔧 后端技术栈（Python）

### FastAPI
- **版本**：`0.104.1`
- **说明**：现代、快速的 Web 框架，用于构建 API
- **选择原因**：
  - 自动生成 Swagger 文档
  - 基于 Python 类型提示，代码更清晰
  - 性能优秀（基于 Starlette 和 Pydantic）
  - 支持异步操作

### Uvicorn
- **版本**：`0.24.0`
- **说明**：ASGI 服务器，用于运行 FastAPI 应用
- **选择原因**：
  - FastAPI 官方推荐的服务器
  - 支持异步和高性能
  - 支持热重载（开发时自动重启）

### PyMySQL
- **版本**：`1.1.0`
- **说明**：MySQL 数据库连接库
- **选择原因**：
  - 纯 Python 实现，无需编译
  - 与现有代码兼容（旧系统也在用）
  - 支持连接池和事务

### Pydantic
- **版本**：`2.5.0`
- **说明**：数据验证库，FastAPI 的核心依赖
- **选择原因**：
  - FastAPI 自动使用 Pydantic 进行数据验证
  - 基于 Python 类型提示
  - 自动生成 JSON Schema

### Python-Multipart
- **版本**：`0.0.6`
- **说明**：处理表单数据（multipart/form-data）
- **选择原因**：
  - FastAPI 处理文件上传时需要
  - 轻量级，性能好

---

## 🎨 前端技术栈（Node.js）

### Vue
- **版本**：`3.3.13`
- **说明**：渐进式 JavaScript 框架
- **选择原因**：
  - 现代化框架，性能优秀
  - 组件化开发
  - 丰富的生态系统

### TypeScript
- **版本**：`5.3.3`
- **说明**：JavaScript 的超集，添加了类型系统
- **选择原因**：
  - 类型安全，减少错误
  - 更好的 IDE 支持
  - 代码更易维护

### Vite
- **版本**：`5.0.8`
- **说明**：下一代前端构建工具
- **选择原因**：
  - 极快的开发服务器启动速度
  - 热模块替换（HMR）速度快
  - 现代化的构建工具

### Element Plus
- **版本**：`2.4.4`
- **说明**：基于 Vue 3 的组件库
- **选择原因**：
  - 组件丰富，开箱即用
  - 文档完善
  - 与 Vue 3 完美兼容
  - 类似 Streamlit 的组件，迁移成本低

### Axios
- **版本**：`1.6.2`
- **说明**：HTTP 客户端库
- **选择原因**：
  - 支持 Promise
  - 支持请求/响应拦截器
  - 自动转换 JSON 数据
  - 浏览器和 Node.js 都支持

### Vue Router
- **版本**：`4.2.5`
- **说明**：Vue.js 官方路由管理器
- **选择原因**：
  - Vue 官方推荐
  - 支持路由守卫（权限控制）
  - 支持动态路由

### ECharts
- **版本**：`5.4.3`
- **说明**：强大的数据可视化库
- **选择原因**：
  - 功能强大，支持多种图表类型
  - 性能优秀
  - 文档完善
  - 可以替代 Streamlit 中的 PyEcharts

### Vue-ECharts
- **版本**：`6.6.1`
- **说明**：ECharts 的 Vue 3 封装
- **选择原因**：
  - 在 Vue 中使用 ECharts 更方便
  - 支持响应式更新
  - 与 Vue 3 兼容
