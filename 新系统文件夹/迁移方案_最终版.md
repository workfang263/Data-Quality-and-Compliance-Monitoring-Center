# Streamlit → Vue + FastAPI 迁移方案（最终版）

## 📋 方案概述

**采用方案二：完全新建系统，复制代码，并行运行**

### 核心原则
1. ✅ **完全隔离**：新旧系统互不影响
2. ✅ **并行运行**：两个系统同时运行，不同端口
3. ✅ **共用数据库**：数据一致，无需迁移
4. ✅ **逐步切换**：功能模块逐步迁移，测试通过后再切换

## 🚀 实施步骤

### 阶段1：搭建FastAPI后端（1-2天）

#### 1.1 创建目录结构
```bash
mkdir backend
cd backend
mkdir -p app/api app/models app/services
```

#### 1.2 复制必要文件
```bash
# 复制 database.py
cp ../database.py app/services/database.py

# 复制 config.py（或新建，只包含数据库配置）
cp ../config.py app/config.py
```

#### 1.3 安装依赖
```bash
cd backend
pip install fastapi uvicorn sqlalchemy pymysql pydantic python-multipart
```

#### 1.4 创建 FastAPI 入口
```python
# backend/app/main.py
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI(title="Shoplazza Dashboard API")

# 配置CORS（允许Vue前端访问）
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],  # Vue开发服务器
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# 导入路由
from app.api import dashboard, owners, stores

app.include_router(dashboard.router)
app.include_router(owners.router)
app.include_router(stores.router)
```

#### 1.5 创建第一个API（看板数据）
```python
# backend/app/api/dashboard.py
from fastapi import APIRouter
from app.services.database import Database

router = APIRouter()
db = Database()

@router.get("/api/dashboard/data")
async def get_dashboard_data(
    shop_domain: str = None,
    start_date: str = None,
    end_date: str = None
):
    # 直接调用复制的 database.py 的方法
    data = db.get_hourly_data_with_spend(...)
    return {"data": data}
```

### 阶段2：搭建Vue前端（2-3天）

#### 2.1 创建Vue项目
```bash
npm create vite@latest frontend -- --template vue-ts
cd frontend
npm install
```

#### 2.2 安装依赖
```bash
npm install element-plus axios vue-router echarts
npm install -D @types/node
```

#### 2.3 配置Axios
```typescript
// frontend/src/api/request.ts
import axios from 'axios'

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000',
  timeout: 30000
})

export default request
```

#### 2.4 创建第一个页面（看板）
```vue
<!-- frontend/src/views/Dashboard.vue -->
<template>
  <div>
    <el-table :data="tableData">
      <!-- 表格内容 -->
    </el-table>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue'
import request from '@/api/request'

const tableData = ref([])

onMounted(async () => {
  const response = await request.get('/api/dashboard/data')
  tableData.value = response.data.data
})
</script>
```

### 阶段4：并行运行（持续）

#### 启动脚本
```bash

streamlit run dashboard.py --server.port 8502

# 终端1：FastAPI

# 终端2：Vue
```

## 🔒 风险控制

### 1. 代码隔离
- ✅ 新系统有自己的 `database.py`，修改不影响旧系统
- ✅ 新系统有自己的 `config.py`，配置独立

### 2. 数据安全
- ✅ 两个系统共用数据库，数据一致
- ✅ 如果新系统写错代码，不会影响数据库（因为只是读数据）

### 3. 回滚方案
- ✅ 保留 Streamlit 代码，随时可以切回
- ✅ 如果新系统有问题，立即切回旧系统

### 4. 错误隔离
- ✅ 新系统出错不影响旧系统
- ✅ 前端可以配置后端地址，随时切换

## 📊 优势总结

### 方案二的优势
1. ✅ **完全隔离**：新旧系统互不影响
2. ✅ **独立开发**：可以重构和优化新系统
3. ✅ **安全切换**：旧系统作为备份，随时可以切回
4. ✅ **代码清晰**：新系统代码独立，易于维护

### 与方案一的对比
| 特性 | 方案一（复用） | 方案二（复制） |
|------|--------------|--------------|
| 代码隔离 | ❌ 耦合 | ✅ 完全隔离 |
| 修改影响 | ⚠️ 可能影响旧系统 | ✅ 不影响旧系统 |
| 代码维护 | ✅ 只需维护一份 | ⚠️ 需要维护两份 |
| 安全性 | ⚠️ 中等 | ✅ 高 |

**结论：方案二更适合"新建系统"的场景**
