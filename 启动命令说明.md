# 系统启动命令说明

## 🆕 新系统（Vue + FastAPI）

### 后端 API（FastAPI）

#### 方式1：使用 uvicorn 命令（推荐）
```bash
# 在项目根目录执行
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 方式2：使用 Python 模块方式
```bash
# 在项目根目录执行
cd backend
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 参数说明：
- `--reload`：开发模式，代码修改后自动重启
- `--host 0.0.0.0`：允许局域网访问（如果只需要本地访问，可以改为 `127.0.0.1`）
- `--port 8000`：API服务端口

#### 访问地址：
- API服务：`http://localhost:8000`
- API文档（Swagger）：`http://localhost:8000/docs`
- API文档（ReDoc）：`http://localhost:8000/redoc`

---

### 前端（Vue + Vite）

#### 启动开发服务器
```bash
# 在项目根目录执行
cd frontend
npm run dev
```

#### 访问地址：
- 前端页面：`http://localhost:5173`

#### 其他命令：
```bash
# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

---

### 同时启动前后端（需要两个终端窗口）

#### 终端1：启动后端
```bash
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 终端2：启动前端
```bash
cd frontend
npm run dev
```

## 🔧 老系统（Streamlit）

### 启动看板

#### 方式1：使用批处理脚本（推荐）
```bash
# 在项目根目录执行
run_dashboard.bat
```

#### 方式2：直接运行 Streamlit 命令
```bash
# 在项目根目录执行
streamlit run dashboard.py --server.port 8502 --server.address 0.0.0.0
```

#### 参数说明：
- `--server.port 8502`：看板服务端口
- `--server.address 0.0.0.0`：允许局域网访问

#### 访问地址：
- 看板页面：`http://localhost:8502`
- 局域网访问：`http://你的IP地址:8502`

## 📋 快速启动脚本（可选）

### 创建新系统启动脚本

#### Windows批处理脚本：`启动新系统.bat`
```batch
@echo off
chcp 65001 >nul 2>&1
echo ========================================
echo 启动新系统（Vue + FastAPI）
echo ========================================
echo.

echo [1/2] 启动后端 API...
start "后端API" cmd /k "cd backend && uvicorn app.main:app --reload --host 0.0.0.0 --port 8000"

timeout /t 3 /nobreak >nul

echo [2/2] 启动前端...
start "前端" cmd /k "cd frontend && npm run dev"

echo.
echo ========================================
echo 启动完成！
echo ========================================
echo.
echo 后端API: http://localhost:8000
echo API文档: http://localhost:8000/docs
echo 前端页面: http://localhost:5173
echo.
echo 按任意键关闭此窗口（服务将继续运行）...
pause >nul
```

#### 创建脚本文件
```bash
# 在项目根目录创建文件：启动新系统.bat
# 复制上面的内容到文件中
```

## ⚠️ 常见问题

### 1. 端口被占用

#### 检查端口占用（Windows）
```bash
# 检查8000端口
netstat -ano | findstr :8000

# 检查5173端口
netstat -ano | findstr :5173

# 检查8502端口
netstat -ano | findstr :8502
```

#### 关闭占用端口的进程
```bash
# 找到进程ID（PID）后，使用任务管理器关闭，或使用命令：
taskkill /PID <进程ID> /F
```

### 2. 依赖未安装

#### 新系统后端依赖
```bash
cd backend
pip install -r requirements.txt
```

#### 新系统前端依赖
```bash
cd frontend
npm install
```

### 3. 数据库连接失败

检查 `backend/config_new.py` 或 `config.py` 中的数据库配置：
- 主机地址
- 端口
- 用户名
- 密码
- 数据库名
