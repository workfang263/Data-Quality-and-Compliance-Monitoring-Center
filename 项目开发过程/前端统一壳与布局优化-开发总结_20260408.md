# 前端统一壳与布局优化 - 开发总结

**文档日期**: 2026-04-08  
**适用范围**: `frontend` 管理后台（看板、负责人汇总、映射编辑、映射操作记录、店铺运营、权限管理）

---

## 一、本次目标

本次开发围绕「统一后台视觉骨架 + 修复页面布局空白」推进，目标如下：

1. 建立统一页面壳标准（容器宽度、背景、页头结构一致）。
2. 将 `StoreOps.vue`、`Dashboard.vue` 接入统一壳。
3. 修复 2 处明显空白问题：
   - 负责人汇总表格右侧空白过大。
   - 看板汇总指标卡片右侧空白（7 卡片未铺满）。
4. 排查并避免双滚动条。

---

## 二、核心改动与原因

### 1) 统一壳标准落地

- 组件基线：
  - `frontend/src/components/PageShell.vue`
  - `frontend/src/components/PageHeaderBar.vue`
- 统一规则：
  - 外层容器: `max-w-[1600px]`
  - 页面背景: `bg-slate-50`
  - 页头结构: 标题 + 副标题 + 右侧操作插槽（actions）

**原因**：将页面“框架层”独立出来，业务页面只关注内容，减少每页重复写外层样式导致的视觉漂移。

### 2) StoreOps 套壳重构（方案 A）

- 文件: `frontend/src/views/StoreOps.vue`
- 改动：
  - 移除私有顶栏与多层包裹（`min-h-screen/max-w-7xl/sticky header`）。
  - 使用 `PageShell` + `PageHeaderBar`。
  - 页面标题与说明文字并入页头。
  - “立即同步”放入页头 actions。

**结果**：与其它后台页在顶部高度、左右留白、背景基线上保持一致。

### 3) Dashboard 套壳统一

- 文件: `frontend/src/views/Dashboard.vue`
- 改动：
  - 使用 `PageShell` 包裹。
  - 新增 `PageHeaderBar title="看板"`。
  - 去掉旧的页面级 `padding/max-width/margin`，改由统一壳控制。

**结果**：看板与店铺运营页在容器宽度与内容起始位置上对齐。

### 4) 双滚动条修复

- 文件: `frontend/src/App.vue`
- 改动：
  - 在 `#app > :last-child` 上增加 `min-height: 0`（保留 `overflow-y: auto`）。

**原因**：Flex 子项默认 `min-height: auto` 可能撑出额外滚动，导致 body 与内容区同时滚动。

### 5) 空白问题修复（两处）

#### 5.1 负责人汇总表格右侧空白

- 文件: `frontend/src/views/Owners.vue`
- 改动：
  - 绝大多数列从固定 `width` 改为 `min-width`（保留负责人列固定）。

**原因**：固定列宽总和小于容器时，会在右侧留下明显空白；改为 `min-width` 后可弹性吃满空间。

#### 5.2 看板汇总卡片右侧空白

- 文件: `frontend/src/components/DashboardSummary.vue`
- 改动：
  - 从 `el-row/el-col` 改为 CSS Grid。
  - 大屏采用 `repeat(7, minmax(0, 1fr))`，确保 7 张卡正好铺满一行。

**原因**：原 `24 栅格 + lg=3` 一行可放 8 格，但仅 7 卡，天然右侧留 1 格空白。

## 三、可见效果（验收口径）

1. 页面外壳统一：背景、最大宽度、页头结构一致。
2. 店铺运营与看板在左右留白、顶部起始位置上对齐。
3. 负责人汇总表格横向观感更平衡，右侧空白显著减少。
4. 看板 7 张汇总卡在大屏一行铺满，不再右侧残留空位。
5. 长页面仅保留主滚动区，不出现明显“双滚动条”。

## 五、后续建议

1. 若继续统一，可将其余页面（如 `Stores.vue`）逐步接入 `PageShell`。

3. 对超宽表格统一策略：默认 `min-width` + 必要列固定，减少空白与挤压同时出现。
