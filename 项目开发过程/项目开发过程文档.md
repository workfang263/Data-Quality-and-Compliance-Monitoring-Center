## 一、项目目标（摘要）

在现有 Shoplazza 数据看板系统上，新增 **店铺运营 · 员工销售额归因** 能力：

- **业务**：按北京时间、按天统计两店（`shutiaoes.myshoplaza.com`、`newgges.myshoplaza.com`）各运营员工的销售额与订单量；数据 **两店不合并**；无法归因金额进 **公共池**，再按「非公共销售额占比」虚拟分摊。
- **技术**：FastAPI + PyMySQL（手写 SQL，无 ORM）+ Vue；店匠 **OpenAPI 2025-06** 与 `.env` 专用 token，与旧 `data_sync` / `shoplazza_stores` 隔离。
- **同步**：`POST /api/internal/store-ops/sync` + **BackgroundTasks**；cron 仅定时 HTTP 触发。
- **权限**：`can_view_store_ops`，与 `can_view_dashboard` 独立。

## 三、本对话窗口中出现的问题与解决方法

### 1. `pip install -r backend/requirements.txt` 报错（UnicodeDecodeError / gbk）

- **现象**：Windows 下 pip 用系统默认编码读 `requirements.txt`，文件中 UTF-8 中文注释导致解码失败。
- **解决**：安装前执行 `$env:PYTHONUTF8=1`（PowerShell），或去掉/英文化注释；成功后依赖已满足。

### 2. Swagger 调用 `POST /api/internal/store-ops/sync` 返回 401

- **现象**：未带鉴权。
- **解决**：在 Swagger 使用 **Authorize** 填入 `Bearer <token>`，或请求头 `X-Internal-Key` 与 `STORE_OPS_SYNC_SECRET` 一致。

### 3. `GET /api/store-ops/report` 返回 422

- **现象**：`start_date` / `end_date` 填成 `4.2`、`4.3` 等非 ISO 日期。
- **解决**：使用 **`YYYY-MM-DD`**（如 `2026-04-02`）；`shop_domain` 不用时留空。

### 4. 同步后库表无数据、日志 SSL + SOCKS 报错

- **现象**：`SSLCertVerificationError`；日志出现 `SOCKSHTTPSConnectionPool`；旧版 `shoplazza_api` 使用 **`verify=False`**，新店匠链路默认 **`verify=True`**，经全局代理时易失败。
- **解决**：
  - 在 **项目根目录 `.env`** 设置 **`STORE_OPS_HTTPS_VERIFY=false`**（或 `0` / `no` / `off`）用于本地开发；生产建议 **`true` 或不写**。
  - **`config_new.py`**：从项目根加载 `.env`，并提供 **`store_ops_https_verify()`**（`os.getenv` 读取，默认开启校验）。
  - **`shoplazza_store_ops_client.py`**：在 `verify=False` 时禁用 `urllib3` 的 `InsecureRequestWarning` 刷屏。
- **说明**：这是 **环境变量 + `os` 读取配置** 的标准做法，不是单独引入新业务逻辑；生产直连公网时默认校验证书即可正常拉取店匠数据。

### 5. Swagger 请求体中 `shop_domains: ["string"]`

- **现象**：占位符被当成真实域名，无法匹配两店与 token。
- **解决**：不传 `shop_domains`（默认两店）或填入真实完整域名数组。

### 6. 前端两店表格不易区分

- **现象**：仅小标题展示域名不够醒目。
- **解决**：在店铺运营页为每个店铺增加 **卡片头「店铺域名」** 区域，突出显示 `shop_domain`（见 `frontend/src/views/StoreOps.vue`）。
