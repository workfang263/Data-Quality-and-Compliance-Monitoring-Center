## 三、窗口内遇到的问题与解决方法

### 问题 1：PowerShell 下 `<` 重定向执行 mysql 脚本报错

- **现象**：`<` 在 PowerShell 中不可用，报 `RedirectionNotSupported`。
- **原因**：PowerShell 与 cmd/bash 重定向语法差异。
- **处理**：
  - 进入 MySQL 后执行 `USE <db>; SOURCE <sql-path>;`；
  - 或改用 `mysql -e "source ..."`。

### 问题 2：MySQL `No database selected`

- **现象**：直接 `DESC users;` 报 `ERROR 1046`。
- **原因**：未先 `USE shoplazza_dashboard;`。
- **处理**：先选库再执行 DDL/DML。

### 问题 3：跨表查询时报 `Illegal mix of collations`

- **现象**：`available-shops` / `available-ad-accounts` 接口 500。
- **原因**：
  - 主系统表使用 `utf8mb4_0900_ai_ci`；
  - 新子系统表使用 `utf8mb4_unicode_ci`；
  - `NOT EXISTS` 子查询做字符串比较时触发排序规则冲突。
- **处理**：
  - 在 SQL 比较处显式加 `COLLATE utf8mb4_unicode_ci`；
  - 接口恢复 200，兼容问题消除。

### 问题 4：Windows PowerShell 中 `curl` 行为异常

- **现象**：`curl` 被解析为 `Invoke-WebRequest`，参数风格不兼容。
- **处理**：
  - 使用 `curl.exe`；
  - 或统一使用 `Invoke-RestMethod` 发请求。

### 问题 5：终端中文显示乱码（接口数据看似异常）

- **现象**：PowerShell 输出中的中文 owner/display_name 乱码。
- **原因**：控制台编码显示问题，不是接口编码问题。
- **处理**：
  - 用 `curl.exe` 输出到 UTF-8 文件再查看，确认 API 返回中文正常；
  - 前端消费 UTF-8 JSON 不受影响。

### 4) 解决过程中的问题

- **PowerShell `Set-Content -Encoding utf8` 破坏中文**：改为 Python `pathlib.write_text(..., encoding='utf-8')` 直接写

- **Windows GBK 终端无法渲染 ✓/✗**：M1 脚本改为 `[Y]` / `[N]`；其他 Python 脚本统一走 `PYTHONIOENCODING=utf-8` 环境变量
