# 数据翻倍问题分析

## 问题现象

- **数据库数据**：订单数 274，销售额 $19154.88
- **API数据**：订单数 137，销售额 $9577.44
- **比例**：数据库数据 = API数据 × 2（正好2倍）

## 可能的原因

### 1. 数据被重复收集 ✅（最可能）

**场景1：运行了fill_today_data.py然后又运行了实时同步**
- `fill_today_data.py` 会清空今天的数据，然后重新收集
- 但如果实时同步脚本也在运行，可能会重复累加

**场景2：fill_date_data.py使用了累加模式**
- `fill_date_data.py` 虽然会先清空数据，但使用的是 `insert_or_update_hourly_data_incremental`（累加模式）
- 如果同一个时间段被处理了两次，就会重复累加

**场景3：实时同步脚本对同一时间段处理了两次**
- 如果实时同步脚本的同步状态有问题，可能对同一时间段的数据重复处理

### 2. 验证脚本的API查询逻辑有问题

验证脚本使用的查询时间范围：
```python
today_end = now.replace(hour=now.hour, minute=now.minute, second=59, microsecond=999999)
```

这可能会导致查询到当前时间之后的数据，但实际上应该只查询到最近完成的5分钟段。

## 检查步骤

1. ✅ 检查数据库内部一致性（明细表 vs 汇总表）- **已检查，一致**
2. ⬜ 检查是否有重复的小时记录 - **已检查，没有重复**
3. ⬜ 检查是否运行了fill_today_data.py或fill_date_data.py
4. ⬜ 检查实时同步脚本的运行状态和同步时间

## 解决方案

### 临时解决方案

如果确定数据被重复收集，可以：
1. 运行 `fill_today_data.py` 重新收集今天的数据（会先清空）
2. 或者运行 `fill_date_data.py today` 重新收集

### 根本解决方案

需要检查：
1. 是否同时运行了多个数据收集脚本
2. 实时同步脚本的同步状态是否正确
3. 是否有定时任务重复执行
