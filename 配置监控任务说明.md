# 数据同步监控任务配置说明

## 🚀 方式一：自动配置（推荐）

### 步骤：

1. **以管理员身份运行 PowerShell**
   - 按 `Win + X`，选择"Windows PowerShell (管理员)"
   - 或者在开始菜单搜索 PowerShell，右键选择"以管理员身份运行"

2. **执行配置脚本**
   ```powershell
   cd "D:\projects\line chart"
   .\setup_db_watcher_task.ps1
   ```

3. **完成！**
   - 脚本会自动创建任务计划
   - 任务名称：`数据同步监控`
   - 执行频率：每 5 分钟执行一次

---

## 🔧 方式二：手动配置

### 步骤：

1. **打开任务计划程序**
   - 按 `Win + R`，输入 `taskschd.msc`，回车

2. **创建基本任务**
   - 右侧点击"创建基本任务"
   - 名称：`数据同步监控`
   - 描述：`监控数据同步脚本运行状态，每5分钟检查一次`

3. **设置触发器**
   - 选择"重复任务"
   - 开始时间：当前时间
   - 重复间隔：每 5 分钟
   - 持续时间：无限期

4. **设置操作**
   - 操作：启动程序
   - 程序或脚本：`D:\projects\line chart\run_db_watcher.bat`
   - 起始于：`D:\projects\line chart`

5. **高级设置**
   - ✅ 勾选"使用最高权限运行"
   - ✅ 勾选"不管用户是否登录都要运行"
   - ✅ 勾选"如果任务失败，重新启动"
   - 重新启动间隔：1 分钟
   - 尝试重新启动最多：3 次

6. **完成**
   - 点击"完成"
   - 可以在任务计划程序中看到新创建的任务

---

## ✅ 验证配置

### 方法 1：查看任务计划程序
1. 打开 `taskschd.msc`
2. 在任务计划程序库中找到 `数据同步监控`
3. 右键任务 → 运行（测试是否正常）

### 方法 2：查看日志
1. 检查日志文件：`D:\projects\line chart\logs\db_watcher.log`
2. 如果任务正常运行，日志文件会不断更新

### 方法 3：手动运行批处理文件
```bash
# 在项目根目录执行
run_db_watcher.bat
```

---

## 📝 相关文件说明

- **`run_db_watcher.bat`** - 批处理文件，被任务计划程序调用
- **`setup_db_watcher_task.ps1`** - PowerShell 自动配置脚本
- **`tests\db_watcher.py`** - Python 监控脚本
- **`logs\db_watcher.log`** - 监控日志文件

---

## 🔍 故障排查

### 问题 1：任务没有执行
- 检查任务计划程序中的任务状态
- 查看"上次运行结果"是否为成功
- 检查批处理文件路径是否正确

### 问题 2：日志文件没有生成
- 检查 `logs` 目录是否存在
- 检查文件权限
- 手动运行批处理文件测试

### 问题 3：没有收到钉钉告警
- 检查 Webhook URL 是否正确
- 手动运行 `python tests\db_watcher.py` 测试
- 查看日志文件中的错误信息

---

## ⚙️ 修改配置

### 修改检查频率
1. 打开任务计划程序
2. 找到 `数据同步监控` 任务
3. 右键 → 属性 → 触发器
4. 修改"重复间隔"

### 修改告警阈值
编辑 `tests\db_watcher.py`，修改：
```python
ALERT_THRESHOLD_MINUTES = 20  # 改为你想要的分钟数
```

---

## 🗑️ 删除任务

### 方法 1：通过任务计划程序
1. 打开 `taskschd.msc`
2. 找到 `数据同步监控` 任务
3. 右键 → 删除

### 方法 2：通过 PowerShell
```powershell
Unregister-ScheduledTask -TaskName "数据同步监控" -Confirm:$false
```
