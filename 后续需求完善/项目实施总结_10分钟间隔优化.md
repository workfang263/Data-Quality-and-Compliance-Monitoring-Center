## 🎯 项目目标

### 核心目标
实施10分钟间隔+覆盖式写入+进程锁优化方案，彻底解决数据翻倍问题，提升系统稳定性和健壮性。

### 问题2：代码缩进问题

#### 问题描述
在添加进程锁功能时，`try...finally` 块的缩进不正确，导致语法错误。

#### 错误代码
```python
if __name__ == '__main__':
    import sys
    
    lock_handle = acquire_lock()
    
try:  # ❌ 缩进错误，应该在 if 块内部
    # ...
```

#### 解决方法
1. **修正缩进规则**：
   - `if __name__ == '__main__':`：0个空格（顶格）
   - `try:` 和 `finally:`：4个空格
   - `try:` 和 `finally:` 块内的代码：8个空格
   - 更深层的嵌套：每层+4个空格

2. **修正后的代码**：
```python
if __name__ == '__main__':
    import sys
    
    lock_handle = acquire_lock()
    
    try:  # ✅ 4个空格缩进
        # ... 8个空格缩进
    finally:  # ✅ 4个空格缩进
        # ... 8个空格缩进
```

3. **结果**：代码缩进正确，语法检查通过

```

```

#### 错误位置
- `data_sync.py` 第1707行：`store_total_sales = existing['total_gmv'] + data['sales']`
- `data_sync.py` 第1825行：`total_sales = existing_data['total_gmv'] + total_sales`

#### 解决方法
在相加前将 `Decimal` 转换为 `float`：

```python
# 修改前（错误）
store_total_sales = existing['total_gmv'] + data['sales']

# 修改后（正确）
existing_gmv = float(existing['total_gmv']) if existing['total_gmv'] is not None else 0.0
existing_orders = int(existing['total_orders']) if existing['total_orders'] is not None else 0
store_total_sales = existing_gmv + data['sales']
```

#### 修改位置
1. 单店铺明细表写入逻辑（第1707-1709行）
2. 汇总表写入逻辑（第1825-1827行）

类型转换问题解决，代码正常运行

## 🔧 技术要点总结

### 1. 进程锁实现

#### 原理
- 使用 Windows 的 `msvcrt.locking()` 实现文件级别的锁
- 非阻塞模式（`LK_NBLCK`）：如果锁不住，立即返回错误
- 锁文件必须保持打开状态，否则锁会失效

#### 关键代码
```python
def acquire_lock():
    script_dir = os.path.dirname(os.path.abspath(__file__))
    lock_file_path = os.path.join(script_dir, "data_sync.lock")
    lock_file = open(lock_file_path, 'a')
    msvcrt.locking(lock_file.fileno(), msvcrt.LK_NBLCK, 1)
    return lock_file  # ⚠️ 必须返回文件句柄
```

### 2. 覆盖式写入实现

#### 原理
- 先查询数据库已有数据
- 合并数据：新总值 = 数据库已有值 + API增量值
- 覆盖写入：使用 `ON DUPLICATE KEY UPDATE`，不带加号

#### 关键代码
```python
# 1. 查询已有数据
existing_data = db.get_hourly_data_by_time(time_hour)

# 2. 合并数据
if existing_data:
    total_sales = float(existing_data['total_gmv']) + new_sales
    total_orders = int(existing_data['total_orders']) + new_orders
else:
    total_sales = new_sales

# 3. 覆盖写入（不带加号，具有幂等性）
db.insert_or_update_hourly_data(time_hour, total_sales, total_orders, ...)
```

### 3. 连续补录实现

#### 原理
- 计算所有缺失的10分钟段
- 依次处理每个缺失的段
- 如果某个段处理失败，停止后续段的处理

#### 关键代码
```python
# 计算所有缺失的段
segments_to_process = []
segment_start = next_segment_start

while segment_start <= recent_10min_end:
    segment_end = segment_start + timedelta(minutes=10) - timedelta(microseconds=1)
    if segment_end <= now:
        segments_to_process.append((segment_start, segment_end))
    segment_start = segment_end + timedelta(seconds=1)
    # 对齐到10分钟边界
    ...

# 依次处理每个缺失的段
for segment_start, segment_end in segments_to_process:
    success = process_ten_minute_segment(segment_start, segment_end, db)
    if success:
        db.update_sync_status('ten_minute_realtime', segment_end, sync_date, 0)
    else:
        break  # 如果失败，停止后续处理
```
