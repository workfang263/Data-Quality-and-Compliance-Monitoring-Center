## 🎯 测试目标

### 核心测试目标

1. **数据一致性**：验证数据库汇总表的数值正确性
2. **并发安全**：验证多线程并发写入时数据不会丢失
3. **防翻倍**：验证数据不会因为重复执行而翻倍

---

## 🧪 测试方法

### 方法1：数据一致性测试（Pytest）

#### 测试原理

- **模拟多个10分钟段的数据同步**
- **验证数据库汇总表的数值等于所有段的总和**
- **确保数据不会翻倍**

#### 测试场景

1. **多段数据一致性测试**
   - 模拟3个10分钟段（15:00-15:09, 15:10-15:19, 15:20-15:29）
   - 每个段有不同的数据
   - 验证数据库汇总表的数值等于3个段的总和

2. **防翻倍测试**
   - 处理同一个10分钟段两次
   - 验证数据不会翻倍（增量累加模式会累加，这是预期的）

#### 运行方式

```bash
# 使用 pytest 运行
pytest tests/test_data_consistency.py -v -s

# 或者直接运行
python tests/test_data_consistency.py
```

#### 测试结果解读

**通过标准：**
- ✅ 数据库汇总表的数值等于所有段的总和
- ✅ 重复执行同一个段，数据会累加（增量累加模式的预期行为）

**失败情况：**
- ❌ 数据不匹配：说明逻辑有问题
- ❌ 数据翻倍：说明累加逻辑有问题

---

### 方法2：并发写入测试

#### 测试原理

- **模拟多个线程同时写入同一个小时的数据**
- **验证数据库的原子累加是否生效**
- **验证是否有数据丢失**

#### 测试场景

- 创建5个线程，同时写入同一个小时的数据
- 每个线程模拟不同的店铺数据
- 验证数据库中的数据等于所有线程数据的总和

#### 运行方式

```bash
python tests/test_concurrent_write.py
```

#### 测试结果解读

**通过标准：**
- ✅ 数据库中的数据等于所有线程数据的总和
- ✅ 没有数据丢失
- ✅ 没有重复累加

**失败情况：**
- ❌ 数据不一致：可能存在数据丢失或重复累加问题
- ❌ 线程执行失败：可能存在并发冲突

## 🔧 测试技术要点

### 1. Mock 数据的使用

#### 为什么使用 Mock 数据？

- **隔离测试**：不依赖真实的 API，测试更稳定
- **可控性**：可以精确控制测试数据
- **速度**：测试运行更快

#### 如何 Mock 数据？

```python
from unittest.mock import patch, MagicMock

# Mock 函数返回值
@patch('data_sync.sync_store_data_for_ten_minutes')
def test_example(mock_sync_store):
    # 设置 mock 返回值
    mock_sync_store.return_value = {
        'success': True,
        'sales': 100.0,
        'orders': 5,
        'visitors': 100
    }
    
    # 执行测试
    result = process_ten_minute_segment(...)
    
    # 验证结果
    assert result == True
```

---

### 2. 固定随机种子

#### 为什么需要固定随机种子？

- **可重复性**：确保每次运行测试结果一致
- **数据一致性**：期望值和实际值使用相同的数据

#### 如何固定随机种子？

```python
import random

def mock_sync_store_data(shop_domain: str, ...):
    # 使用店铺域名作为种子，确保同一个店铺每次返回相同的数据
    random.seed(hash(shop_domain) % 10000)
    sales = random.uniform(50.0, 200.0)
    orders = random.randint(1, 10)
    visitors = random.randint(10, 100)
    return {...}
```

### 3. 测试数据清理

#### 为什么需要清理测试数据？

- **隔离测试**：避免测试之间相互影响
- **数据准确性**：确保测试结果准确

#### 如何清理测试数据？

```python
@pytest.fixture(autouse=True)
def setup(self):
    # 测试前清理
    self._cleanup_test_data()
    
    yield
    
    # 测试后清理
    self._cleanup_test_data()

def _cleanup_test_data(self):
    with self.db.get_connection() as conn:
        with conn.cursor() as cursor:
            cursor.execute(
                "DELETE FROM shoplazza_overview_hourly WHERE time_hour = %s",
                (self.test_time_hour,)
            )
            conn.commit()
```

## 🔍 问题排查技巧

### 1. 数据不一致问题

**排查步骤：**
1. 检查期望值计算是否正确
2. 检查实际值计算是否正确
3. 检查 Mock 数据是否一致
4. 检查数据库中的数据是否正确

### 2. 并发问题

**排查步骤：**
1. 检查线程是否都成功执行
2. 检查数据库的原子操作是否正常
3. 检查是否有数据丢失
4. 检查是否有重复累加

### 3. Mock 数据问题

**排查步骤：**
1. 检查 Mock 函数是否正确设置
2. 检查随机种子是否固定
3. 检查 Mock 返回值是否一致
