## 📋 第一步：提取公共函数 `_get_order_beijing_time()`

### 1. 函数设计

**位置：** `data_sync.py` 第147行之后（在 `_is_cod_order()` 函数之后）

**函数签名：**
```python
def _get_order_beijing_time(order: Dict[str, Any]) -> Optional[datetime]:
    """
    从订单对象中解析 placed_at 时间，并转换为北京时间（无时区）
    
    【测试要点】
    - 测试UTC时间（带Z）的解析
    - 测试UTC时间（+00:00）的解析
    - 测试北京时间（+08:00）的解析
    - 测试无时区信息的解析
    - 测试缺失placed_at字段的情况
    - 测试时区转换的正确性
    
    Args:
        order: 订单对象，必须包含 placed_at 字段（或 updated_at/created_at 作为后备）
        
    Returns:
        北京时间（datetime对象，无时区信息），如果解析失败返回None
        
    Raises:
        不抛出异常，解析失败时返回None并记录日志
    """
```

### 2. 代码实现要点

**关键点：**
1. **类型一致性**：必须返回无时区的datetime（`replace(tzinfo=None)`）
2. **错误处理**：解析失败时返回None，不抛出异常
3. **后备机制**：如果没有 `placed_at`，尝试 `updated_at` 或 `created_at`

## ⚠️ 潜在风险点

1. **类型不一致**：如果返回带时区的datetime，会导致混合类型比较错误
2. **时区转换错误**：UTC转北京时间需要加8小时
3. **后备机制失效**：如果所有时间字段都缺失，应该返回None
