# 初始化时区配置 - SQL命令详细解释

## 步骤2：初始化时区配置

### 命令1：为所有负责人设置默认时区（UTC+8）

```sql
INSERT INTO owner_timezone_mapping (owner, timezone, timezone_offset) 
SELECT DISTINCT owner, 'Asia/Shanghai', 8.0
FROM store_owner_mapping
WHERE owner IS NOT NULL
ON DUPLICATE KEY UPDATE 
  timezone = VALUES(timezone), 
  timezone_offset = VALUES(timezone_offset);
```

**命令分解：**

1. **`INSERT INTO owner_timezone_mapping (owner, timezone, timezone_offset)`**
   - `INSERT INTO`：插入数据到表中
   - `owner_timezone_mapping`：目标表名
   - `(owner, timezone, timezone_offset)`：要插入的字段列表

2. **`SELECT DISTINCT owner, 'Asia/Shanghai', 8.0 FROM store_owner_mapping`**
   - `SELECT DISTINCT owner`：从 `store_owner_mapping` 表中查询所有不重复的负责人
   - `'Asia/Shanghai'`：固定值，表示时区名称（北京时间）
   - `8.0`：固定值，表示时区偏移量（UTC+8）
   - 作用：为每个负责人生成一条记录，时区都是 UTC+8

3. **`WHERE owner IS NOT NULL`**
   - 过滤条件：只选择负责人不为空的记录
   - 作用：排除没有负责人的店铺

4. **`ON DUPLICATE KEY UPDATE ...`**
   - `ON DUPLICATE KEY`：如果插入时遇到唯一键冲突（owner已存在）
   - `UPDATE timezone = VALUES(timezone)`：更新时区字段为新值
   - `UPDATE timezone_offset = VALUES(timezone_offset)`：更新时区偏移量字段为新值
   - 作用：如果负责人已存在，就更新；如果不存在，就插入

**整体作用：**
- 从 `store_owner_mapping` 表中获取所有负责人
- 为每个负责人插入一条时区配置记录，默认都是 UTC+8（北京时间）
- 如果负责人已存在，就更新为 UTC+8

**预期结果：**
- 所有负责人都会被插入到 `owner_timezone_mapping` 表中
- 时区都是 `Asia/Shanghai`，偏移量都是 `8.0`

---

### 命令2：设置"小一"的时区为UTC-8

```sql
INSERT INTO owner_timezone_mapping (owner, timezone, timezone_offset) 
VALUES ('小一', 'America/Los_Angeles', -8.0)
ON DUPLICATE KEY UPDATE 
  timezone = VALUES(timezone), 
  timezone_offset = VALUES(timezone_offset);
```

**命令分解：**

1. **`INSERT INTO owner_timezone_mapping (owner, timezone, timezone_offset)`**
   - 插入数据到 `owner_timezone_mapping` 表

2. **`VALUES ('小一', 'America/Los_Angeles', -8.0)`**
   - `'小一'`：负责人名称
   - `'America/Los_Angeles'`：时区名称（美国洛杉矶时区，UTC-8）
   - `-8.0`：时区偏移量（UTC-8，比UTC慢8小时）

3. **`ON DUPLICATE KEY UPDATE ...`**
   - 如果"小一"已存在（因为上一步已经插入了），就更新
   - 如果不存在，就插入

**整体作用：**
- 将"小一"的时区从 UTC+8 更新为 UTC-8
- 覆盖上一步设置的默认值

**预期结果：**
- "小一"的时区配置会被更新为：
  - `timezone = 'America/Los_Angeles'`
  - `timezone_offset = -8.0`

## 执行后的验证

执行完这两个SQL后，可以运行以下查询验证：

```sql
-- 查看"小一"的时区配置
SELECT owner, timezone, timezone_offset 
FROM owner_timezone_mapping 
WHERE owner = '小一';
```

**预期结果：**
```

| owner | timezone            | timezone_offset |
+-------+---------------------+----------------+

+-------+---------------------+----------------+
```


## 总结

这两个SQL的作用：

1. **第一个SQL**：为所有负责人设置默认时区 UTC+8（北京时间）
2. **第二个SQL**：将"小一"的时区更新为 UTC-8（洛杉矶时间）

执行后：
- ✅ 所有负责人都有时区配置
- ✅ "小一"的时区是 UTC-8
