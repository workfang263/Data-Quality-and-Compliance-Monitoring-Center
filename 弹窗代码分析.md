## 3. 关键问题分析

### 问题 1：浮层默认隐藏
**CSS（第 1891-1908 行）：**
```css
#owner-modal-overlay {
    position: fixed;
    inset: 0;
    z-index: 9999;
    display: flex;
    align-items: center;
    justify-content: center;
    padding: 16px;
    pointer-events: none;  /* ❌ 默认不可点击 */
    opacity: 0;            /* ❌ 默认透明 */
    transition: opacity 300ms ease-in-out;
}

/* 显示状态：遮罩层可见 */
#owner-modal-overlay.show {
    opacity: 1;            /* ✅ 需要添加 show 类才显示 */
    pointer-events: auto;  /* ✅ 需要添加 show 类才可点击 */
}
```

**问题：** 浮层默认是隐藏的，需要 JavaScript 添加 `show` 类才能显示。

### 问题 2：JavaScript 执行时机
**JavaScript（第 2362-2377 行）：**
```javascript
// 页面加载时初始化（立即执行 + 延迟执行，确保能显示）
(function() {
    // 立即尝试打开（使用 requestAnimationFrame 确保 DOM 已渲染）
    requestAnimationFrame(function() {
        setTimeout(openModal, 10);
    });
    
    // 延迟初始化图表（确保 Streamlit 渲染完成）
    setTimeout(function() {
        if (document.readyState === 'loading') {
            document.addEventListener('DOMContentLoaded', initModalAndChart);
        } else {
            initModalAndChart();
        }
    }, 200);
})();
```

**问题：**
1. 这是一个**立即执行函数（IIFE）**，会在脚本加载时立即执行
2. 但是 `st.markdown` 插入的 HTML 在每次 `st.rerun()` 后可能被重新渲染
3. 如果 Streamlit 清理了 DOM，JavaScript 可能不会重新执行

### 问题 3：openModal 函数（第 2159-2179 行）
```javascript
function openModal() {
    const overlay = document.getElementById('owner-modal-overlay');
    if (!overlay) {
        console.error('未找到浮层元素');
        return;
    }
    
    // 1. 先设置 shouldRender（确保 DOM 存在）
    window.ownerModalState.shouldRender = true;
    overlay.style.display = 'flex';
    
    // 2. 延迟 10ms 设置 isVisible（触发动画）
    if (window.ownerModalState.openTimeout) {
        clearTimeout(window.ownerModalState.openTimeout);
    }
    window.ownerModalState.openTimeout = setTimeout(function() {
        window.ownerModalState.isVisible = true;
        overlay.classList.add('show');  // ✅ 这里添加 show 类
        console.log('弹窗已显示');
    }, 10);
}
```

**问题：**
- 如果 `overlay` 元素不存在，函数会直接返回
- 如果 JavaScript 没有执行，`show` 类永远不会被添加

## 4. 可能的原因

### 原因 1：JavaScript 没有执行
- `st.markdown` 插入的 `<script>` 标签可能被 Streamlit 过滤或清理
- 或者脚本执行时 DOM 还没有准备好

### 原因 2：DOM 元素没有正确插入
- 浮层可能被插入到错误的位置
- 或者被 Streamlit 的容器限制

### 原因 3：z-index 不够高
- Streamlit 某些元素可能 z-index 更高（虽然 9999 应该足够）

### 原因 4：CSS 样式被覆盖
- Streamlit 的默认样式可能覆盖了自定义样式

## 6. 解决方案

### 方案 1：使用 `st.dialog`（推荐）
由于你的 Streamlit 版本是 1.51.0，完全支持 `st.dialog`，建议改用原生弹窗：

```python
@st.dialog("负责人小时趋势")
def show_owner_modal(owner_name, start_date, end_date):
    # 获取数据
    hourly_data = db.get_owner_hourly_data(owner_name, start_date, end_date)
    # ... 渲染图表和数据 ...
    
    if st.button("关闭"):
        st.rerun()

# 在按钮点击时调用
if st.button(owner_name, key=f"owner_click_{idx}"):
    show_owner_modal(owner_name, popup_start_dt, popup_end_dt)
```

### 方案 2：改进当前实现
如果必须使用 `st.markdown`，需要：
1. 提高 z-index 到 99999
2. 添加 `!important` 确保样式优先级
3. 使用内联立即执行脚本，确保每次 rerun 后都执行
4. 添加调试日志

### 方案 3：使用 `st.components.v1.html`
可以更好地控制 DOM 和 JavaScript 执行。
